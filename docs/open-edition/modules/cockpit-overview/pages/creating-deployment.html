Create a deployment package

You have two options:

  1. Create a deployment package and transferring it by selecting a route.

    When using routes, you can transfer your package to many targeted systems with one transfer process.

  2. Create a deployment package and transferring it by selecting a system.

Prerequisites

  • Before you start a deployment process, make sure that you set up a remote system you want to send the deployment packages to. How to set up a remote system, see Remote System Setup.

  • You have selected a deployment authentication in settings → customization component.

    Deployment can take place through Routes or with systems.

Procedure

  1. Open Neptune DXP - Open Edition cockpit.

  2. Go to Deployment, and click Deployment Create.

    Result: The deployment create window opens

Create a deployment package via route

  1. Click Routes

  2. On the list, check one or more routes.

  3. Click Start.

    Result: The deployment create dialog opens.

  4. To select already existing packages, click Packages.

    The number next to Packages indicates the number of available packages.

  5. Check the package or packages you want to deploy.

  6. To select a service you want to deploy artefacts from, click the arrow next to the service group. In the example, we selected Launchpad in the Administration tab.

    The number next to the service indicates the number of available artefacts.

    Result Deployment create shows a list of all artefacts of the selected service.

  7. Check the artefacts you want to add.

  8. To …​. , check Include relations.

  9. Click Create.

    Result: The Deploy window opens.

  10. Verify the package(s), and their artefacts you want to deploy. Check or uncheck items to adjust your selection.

  11. Click Create.

    Result: Deployment Create sends the packages to the remote system for approval.

Create a deployment package via system

  1. Click System

  2. On the list, check the system or systems you want to create a deployment package for.

  3. Click Start.

    Result: The deployment create dialog opens.

  4. Click the arrow next to the service group, and select a service you want to deploy artefacts from. In the example, we selected Launchpad in the Administration tab.

    The number next to the service indicates the number of available artefacts.

    Result Deployment create shows a list of all artefacts of the selected service.

  5. Check the artefacts you want to add.

  6. Click Create.

    Result: The Deploy window opens.

  7. Verify the package(s), and their artefacts you want to deploy. Check or uncheck items to adjust your selection.

  8. Click Create.

    Result: Deployment Create creates your deployment package, and will ask you for approval or transfer according to the deployment authentication you selected before.
